üîê API Rules Reference (COMPLETE SYNTAX)

Rule Values

""     // Empty string = PUBLIC ACCESS (anyone can perform action)
null   // null = ADMIN ONLY (only admin users can perform action)
"rule" // String = CUSTOM RULE (filter expression, see examples below)

Public Access Rules

{
  "listRule": "",      // Anyone can list/read records
  "viewRule": "",      // Anyone can view individual records
  "createRule": "",    // Anyone can create records
  "updateRule": "",    // Anyone can update records
  "deleteRule": ""     // Anyone can delete records
}

Admin Only Rules

{
  "listRule": null,    // Only admin can list records
  "viewRule": null,    // Only admin can view records
  "createRule": null,  // Only admin can create records
  "updateRule": null,  // Only admin can update records
  "deleteRule": null   // Only admin can delete records
}

Custom Filter Rules Examples

// User can only access their own records
"@request.auth.id = user_id"

// Only published records are visible
"status = 'published'"

// Multiple conditions (AND)
"status = 'published' && user_id = @request.auth.id"

// Multiple conditions (OR)
"status = 'published' || user_id = @request.auth.id"

// Date-based rules
"created >= @todayStart && created <= @todayEnd"

// Field existence checks
"email != ''"

// Relation-based rules
"@request.auth.role.name = 'admin'"

Available Request Variables

@request.auth.id          // Current user ID
@request.auth.email       // Current user email
@request.auth.*           // Any user field
@request.data.*           // Submitted form data
@request.headers.*        // Request headers
@todayStart              // Today 00:00:00
@todayEnd                // Today 23:59:59
@now                     // Current timestamp

üöÄ Setup Process Guide

  1. Configure Connection
    • Set PocketBase URL (default: /)
    • Set admin email and password
    • Click "Test Connection" to verify
  2. Define Schema
    • Edit collections JSON in left panel
    • Use proper field types and validation
    • Set appropriate API rules
  3. Prepare Seed Data
    • Edit seed data JSON in right panel
    • Match collection names exactly
    • Use valid field values
  4. Run Setup
    • "Schema Only" - Create collections without data
    • "Seed Only" - Add data to existing collections
    • "Full Setup" - Schema + Seed in one step
  5. Validate
    • Run validation to check data integrity
    • Verify all collections and records
    • Check for orphaned relations
